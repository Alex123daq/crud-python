# crud-python
Proyecto CRUD simple en Python para práctica de Git Flow.


Este proyecto es un CRUD sencillo hecho en Python, organizado para cumplir con los requisitos de un flujo de trabajo Git Flow. El objetivo principal no es la complejidad del programa, sino demostrar un manejo correcto de ramas, pull requests y control de versiones.

Descripción del proyecto

La lógica se explica paso por paso mediante comentarios, donde se describe:

Cómo se crearía un registro (Create)
Cómo se leerían los datos (Read)
Cómo se actualizaría un registro (Update)
Cómo se eliminaría un dato (Delete)

Y una parte final donde estaría un menú general que agruparía todo el CRUD

El proyecto usa una lista como estructura básica para almacenar la información, ya que lo importante es mostrar el proceso de desarrollo con Git Flow.

Tecnologías utilizada:
Python (sin frameworks, sin bases de datos)
Git
GitHub


El proyecto utiliza un flujo Git Flow simplificado con tres ramas principales:
main
dev
qa 

Y además se crearon 5 ramas feature, cada una dedicada a una parte del CRUD:
feature/crud-create
feature/crud-read
feature/crud-update
feature/crud-delete
feature/menu-system

Se realizaron cambios dentro de la rama
Se hicieron commits
Se generaron tres Pull Requests:

uno hacia dev
uno hacia qa
uno hacia main
Todos los PR fueron cerrados correctamente.
Total de PR completados: 15
